Getting Started

Overview

KiRePa is a full-stack Wi-Fi hotspot management system. Customers purchase data plans via M-Pesa, receive voucher codes via SMS, and connect to the internet. The system includes Kopa (data borrowing) and a reward points program.

💳

M-Pesa Payments

STK Push via Safaricom Daraja API. Payments trigger automatic voucher creation.

🎫

Voucher System

Auto-generated 12-character codes sent via SMS after successful payment.

🤝

Kopa Feature

Customers borrow data now and repay automatically on their next purchase.

⭐

Reward Points

Earn 1 point per 10 Ksh spent. Redeem for free data vouchers.

📱

OTP Verification

SMS-based OTP for all sensitive actions via Africa's Talking.

📊

Admin Dashboard

Dark-themed panel with revenue charts, customer stats, and transaction logs.

Follow these steps to run the project locally on your computer.

Step-by-step setup

1

Clone the repository

Download the project from GitHub

bash
git clone https://github.com/patomurgong/final-hotspot-system.git
cd final-hotspot-system
Windows PowerShell
# Create and activate virtual environment
python -m venv .venv
.venv\Scripts\activate

# Install all dependencies
pip install -r requirements.txt
.env file (create this in the project root)
SECRET_KEY=your-secret-key-here
DEBUG=True
NGROK_URL=https://your-ngrok-url.ngrok-free.app
AFRICASTALKING_USERNAME=sandbox
AFRICASTALKING_API_KEY=your-at-api-key
AFRICASTALKING_SENDER_ID=KIREPA
REDIS_URL=redis://127.0.0.1:6379/0
Run the server
python manage.py migrate
python manage.py createsuperuser
python manage.py runserver 0.0.0.0:8002
✅ Done! Visit http://127.0.0.1:8002 for the portal and http://127.0.0.1:8002/admin for the admin dashboard.

Deployment on Railway

1

Create Railway account

Go to railway.app and sign up with your GitHub account

2

New Project → GitHub Repository

Select final-hotspot-system from your repos

3

Add PostgreSQL + Redis

Click + New → Database → Add PostgreSQL, then Redis

4

Set environment variables

Add SECRET_KEY, DEBUG=False, MPESA credentials, and AT credentials in Railway Variables tab

5

Run migrations

In Railway Shell tab: python manage.py migrate && python manage.py createsuperuser

User Guide

Buying a Plan

1️⃣

Select a plan

Scroll down to "Purchase a Plan" and tap any circle plan button. Choose the data size that fits your needs.

2️⃣

Enter your M-Pesa number

Type your Safaricom number in the format 07XXXXXXXX (10 digits)

3️⃣

Pay with M-Pesa

Click "Pay with MPESA". An STK push prompt will appear on your phone. Enter your M-Pesa PIN.

4️⃣

Receive voucher via SMS

Within seconds of payment confirmation, you'll receive an SMS with your voucher code.

5️⃣

Activate and browse

Click "Enter Voucher", enter your phone number and voucher code, verify OTP, and you're connected!

User Guide

Using a Voucher

After receiving your voucher code via SMS:

📱

Click "Enter Voucher"

Find this button at the top of the portal

🔢

Enter phone + voucher code

Use the same phone number you paid with. The voucher code is in your SMS.

🔐

Verify OTP

An OTP will be sent to your phone. Enter all 6 digits within 5 minutes.

✅

Connected!

Your device is now connected to the internet. The page will redirect to Google automatically.

ℹ️ Lost your voucher? Use the "Lookup Voucher" button to retrieve all vouchers linked to your phone number.

Technical Docs

Payment Flow

What happens when a customer pays:

1️⃣

Frontend calls /api/initiate-mpesa-payment/

Checks for unpaid Kopa first. If clear, sends STK Push to Safaricom.

2️⃣

MpesaTransaction saved as PENDING

Stores CheckoutRequestID to match the callback later.

3️⃣

Customer enters M-Pesa PIN on phone

Safaricom processes the payment on their side.

4️⃣

Safaricom calls /api/mpesa/callback/

Our server receives confirmation with receipt number and amount.

5️⃣

Atomic transaction block runs

1. Updates transaction to COMPLETED   2. Repays any Kopa   3. Creates voucher   4. Awards points

6️⃣

SMS sent to customer

Voucher code, plan details, and points earned notification all sent via Africa's Talking.

✅ All steps are atomic If any step fails (e.g. voucher creation error), the database transaction rolls back and no partial state is saved. This prevents double-charging or vouchers being created without payment.
